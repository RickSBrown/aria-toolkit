checkAriaOwns(element) → {Array.<ValidationResult>}

Check the implementation of the "aria-owns" attribute on this element. This includes checking that:
  • the element MUST not "aria-own" an id that is "aria-owned" somewhere else
  • the element it "aria-owns" actually exists in the DOM
  • the element SHOULD not contain the element it "aria-owns"
Parameters:
Name Type Description
element Element A DOM element to check.
Source:
  • aria.validator.js, line 228
Returns:
Any failures or warnings.
Type
Array.<ValidationResult>
Example
var element = document.createElement("div");
element.setAttribute("aria-owns", "kungfu");
element = element.appendChild(document.createElement("span"));
element.setAttribute("id", "kungfu");
aria.checkAriaOwns(element);//this will flag a warning because the element "aria-owns" an element it implicitly owns in the DOM.

checkIds(element) → {Array.<ValidationResult>}

Checks the implementation of IDs within the context of this element. This is not a true ARIA check however ARIA depends on IDs being correctly implemented.
This check looks for:
  • Duplicate IDs
  • IDs with illegal characters (according to HTML5 rules).
Parameters:
Name Type Description
element Element The element which scopes the check. This REALLY should be a document element (node type 9), it would be stupid to pass in anything else except for unit testing purposes.
